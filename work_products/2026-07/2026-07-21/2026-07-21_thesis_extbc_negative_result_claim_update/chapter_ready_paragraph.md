The external-boundary implementation should be reported as a negative but
scientifically useful result. The current runtime path accepts setup-side
ambient-wall metadata and executes on the Salt2 training case, and the Salt1
ambient-wall recovery closes an important train-envelope metadata gap. However,
the same evidence does not support thermal admission: the train-only diagnostic
run retains large temperature residuals, with the dominant error localized to
the heated-incline TW5/TW6 region, and heater/cooler/test-section source/sink
terms remain provenance candidates rather than admitted runtime source models.
Thus the thesis can claim implementation readiness and improved provenance
separation, but must not claim final predictive accuracy, source/sink runtime
admission, validation/holdout performance, or a frozen candidate from this path.
