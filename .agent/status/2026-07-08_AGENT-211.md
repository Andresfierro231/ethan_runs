## Inferred Interpretation

- The cooler/HX duty magnitude remains the dominant thermal mismatch.  Replacing
  only the predictive HX duty with CFD cooler wallHeatFlux nearly closes loop
  delta-T and cuts mean-temperature error by about an order of magnitude.
- Heater wallHeatFlux and test-section sink corrections are physically
  important but cannot be stacked naively with imposed cooler duty until the 1D
  model has a temperature-dependent passive boundary network.
- Full fixed-Q heat-ledger replay is useful for energy-balance diagnostics but
  under-anchors absolute mean temperature in a periodic 1D thermal solve.
- First-class Fluid fixed-mdot metadata is still needed so reporting layers
  cannot mix thermal replay rows with predictive mdot rows.

## Blockers

- `AGENT-210` actively owns the external Fluid solver files, so this task did
  not edit `solver.py`.
- Fluid currently lacks a first-class `fixed_mdot_kg_s` / hydraulic mode field.
- Fluid prescribed-loss semantics apply a supplied loss map globally, so
  prescribing only the quartz sink also suppresses unlisted passive losses.
- No CFD `qr` radiation field exists for Salt 2/3/4; radiation cannot be
  separated from net `wallHeatFlux` without a new diagnostic/export.
